# เริ่มต้นใช้งาน RoundTable Framework

ยินดีต้อนรับสู่ **RoundTable** — ระบบจัดการทีม AI แบบหลายทีมสำหรับ Claude Code

คู่มือนี้จะพาคุณเริ่มใช้งานได้ภายใน 5 นาที

---

## สิ่งที่คุณเพิ่งติดตั้ง

RoundTable จัดระเบียบการทำงานของ Claude Code ให้เป็น **ทีมที่มีบทบาทชัดเจน** มีระบบบันทึก log อัตโนมัติ และ workflow ที่เป็นระบบ เปรียบเสมือน "สตูดิโอพัฒนาซอฟต์แวร์เสมือน" ที่แต่ละทีม AI มีหน้าที่เฉพาะ:

| ทีม | บทบาท | เหมาะกับงาน |
|-----|--------|-------------|
| **Overseer** | บริหารโปรเจค, ประสานงาน | สถาปัตยกรรม, วางแผน, มอบหมายงาน |
| **Monolith** | Backend, โครงสร้างพื้นฐาน | ฐานข้อมูล, Cloud, เอกสาร |
| **Syndicate** | API, ปรับแต่ง, ความปลอดภัย | Integration, Query tuning, Security |
| **Arcade** | Frontend, งานสร้างสรรค์ | UI, Gamification, ออกแบบ |
| **Cipher** | นิติวิทยาศาสตร์ดิจิทัล | วิเคราะห์ฮาร์ดแวร์, กู้ข้อมูล |

---

## เซสชันแรกของคุณ (5 นาที)

### ขั้นตอนที่ 1: เปิด Claude Code ในโปรเจคของคุณ

ตรวจสอบว่าคุณอยู่ใน workspace ที่ติดตั้ง RoundTable แล้ว จะเห็นโฟลเดอร์ `.claude/`

### ขั้นตอนที่ 2: เริ่มสนทนากับ Claude Code

Claude Code จะอ่าน `.claude/CLAUDE.md` อัตโนมัติและเข้าสู่ระบบ RoundTable เริ่มพิมพ์ได้เลย:

```
You: สวัสดีครับ เริ่มงานวันนี้
```

Claude จะเริ่มต้นเป็น **Team Overseer (AM)** เปิด log ประจำวัน และรอคำสั่งจากคุณ

### ขั้นตอนที่ 3: ตั้งค่าโปรเจค

บอก Claude ให้ตั้งค่าโปรเจค:

```
You: ตั้งค่า ProjectEnvironment สำหรับโปรเจค MyApp mode Centralized
```

หรือภาษาอังกฤษ:

```
You: Set up ProjectEnvironment for project MyApp in Centralized mode
```

คำสั่งนี้จะอัพเดท `.claude/ProjectEnvironment.md` ซึ่งเป็นไฟล์ที่ทุกทีมใช้อ้างอิงตำแหน่งโค้ดของคุณ

### ขั้นตอนที่ 4: สั่งงานแรก

บอก Claude ว่าต้องการอะไร:

```
You: วิเคราะห์โครงสร้างโค้ดปัจจุบันให้หน่อย
```

```
You: Analyze the current codebase structure
```

Claude (ในบทบาท Overseer) จะส่งงานไปยังทีมที่เหมาะสม สร้างแผนงาน และขออนุมัติก่อนลงมือทำ

## คำสั่งลัด (Slash Commands)

คำสั่งเหล่านี้ทำให้ workflow อัตโนมัติ:

| คำสั่ง | ทำอะไร | ใช้เมื่อไหร่ |
|--------|--------|-------------|
| `/roundtable-open` | เปิด log เซสชันใหม่ | ทุกครั้งที่เริ่มสนทนา |
| `/ticket-create MON-01 MyTask` | สร้าง ticket งาน | เมื่อมอบหมายงานเฉพาะ |
| `/briefing-create Monolith 1` | สร้างบรีฟงานให้ทีม | เมื่อเริ่ม phase ใหม่ |
| `/overseer-report MON-01` | รายงานผลงาน | เมื่องานเสร็จ |
| `/audit MyProject` | ตรวจสอบโค้ดแบบครบทุกมิติ | เมื่อต้องการ review โค้ดอย่างละเอียด |
| `/l3-scan MyProject` | สแกนโค้ดเชิงลึก | เมื่อรับโปรเจคที่มีโค้ดอยู่แล้ว |

## หลักการทำงาน

### กฎ "วางแผนก่อนทำ"

เมื่อคุณสั่งให้ Claude สร้างอะไร Claude จะ:

1. **สร้างแผนงานก่อน** (ไม่กระโดดไปเขียนโค้ดเลย)
2. **นำเสนอแผนให้คุณอนุมัติ**
3. **ลงมือทำหลังจากคุณสั่ง "ทำเลย"**

ป้องกันการทำงานที่ไม่ตรงตามต้องการ

### ระบบบันทึก Log

ทุกเซสชันจะถูกบันทึกใน `RoundTable/DD-MM-YYYY_RoundTable.md` ทำให้:

- มีประวัติการตัดสินใจทั้งหมด
- ติดตามได้ว่าใครทำอะไร เพราะอะไร
- ส่งต่องานระหว่างเซสชันได้ง่าย (แม้หลัง `/compact`)

### การส่งต่องานอัตโนมัติ

ไม่ต้องจำว่าทีมไหนทำอะไร แค่อธิบายงาน Overseer (AM) จะส่งต่อให้:

- *"แก้ API login"* → Syndicate
- *"ออกแบบ dashboard ใหม่"* → Arcade
- *"สร้างโครงสร้างฐานข้อมูล"* → Monolith
- *"ตรวจสอบดิสก์ที่เสีย"* → Cipher

## โหมดโปรเจค

| โหมด | ใช้เมื่อ |
|------|---------|
| **Centralized** | แผนงานและโค้ดอยู่ใน repo เดียวกัน |
| **Decentralized** | แผนงานแยกจากโค้ด (เช่น มีหลาย repo) |

โปรเจคทั่วไปแนะนำใช้ **Centralized**

## โครงสร้างไฟล์

```
your-project/
├── .claude/
│   ├── CLAUDE.md              ← กฎหลักของระบบ (โหลดอัตโนมัติ)
│   ├── ProjectEnvironment.md  ← ทะเบียนโปรเจค
│   ├── settings.json          ← ตั้งค่า Hook
│   ├── policies/              ← ไฟล์นโยบาย 7 ไฟล์ (โหลดตามต้องการ)
│   ├── Team Roster/           ← คำจำกัดความบทบาททีม
│   ├── agents/                ← Subagent definitions
│   ├── skills/                ← คำสั่งลัดอัตโนมัติ
│   └── Team Chat/             ← บันทึกรายวันของทีม
├── RoundTable/                ← บันทึกเซสชันหลัก
└── Development/               ← เอกสารแผนงานและ implementation logs
```

## เคล็ดลับ

1. **คุณคือ Commander ท่านผู้บัญชาการ** — ทุกทีมรายงานต่อคุณ คุณมีอำนาจสูงสุด
2. **อนุมัติแผนก่อนทำ** — Claude จะถามเสมอ ตอบ "ทำเลย" หรือ "ไปต่อ" เพื่ออนุมัติ
3. **ใช้ภาษาไทยหรืออังกฤษได้** — RoundTable รองรับทั้งสองภาษา
4. **ไม่ต้องห่วงเรื่อง log** — Claude จัดการบันทึกให้อัตโนมัติ
5. **เริ่มง่ายๆ** — ไม่ต้องใช้ทุกทีมตั้งแต่วันแรก Overseer ทีมเดียวจัดการงานส่วนใหญ่ได้
